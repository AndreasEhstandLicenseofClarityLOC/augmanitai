# AUGMANITAI Trademark Policy

## The AUGMANITAI Name and Mark

"AUGMANITAI" is a project name and word mark used to identify the terminology compendium and associated tooling authored by Andreas Ehstand. The name is also used to identify the published academic work registered under DOI 10.5281/zenodo.19481331.

This trademark policy describes the permitted and prohibited uses of the AUGMANITAI name, logos, and other identifying marks. It is based on the Apache Software Foundation trademark policy and the Mozilla trademark policy, adapted for a small academic project.

## Permitted Uses (No Permission Required)

You may use the AUGMANITAI name **without asking** for any of the following:

1. **Referring to the project** — You can say "I use AUGMANITAI", "AUGMANITAI terminology", "the AUGMANITAI Python package", etc.

2. **Factual description** — You can accurately describe what AUGMANITAI is, who created it, and what it contains.

3. **Academic citation** — You can cite AUGMANITAI in research papers, dissertations, books, and other scholarly work using the name and the canonical citation format.

4. **News and commentary** — You can write about AUGMANITAI in blog posts, news articles, reviews, and criticism.

5. **Interoperability references** — If your tool reads, writes, or queries AUGMANITAI data, you can say "Compatible with AUGMANITAI" or "Implements the AUGMANITAI schema", provided the claim is accurate.

6. **Teaching and training** — You can use the AUGMANITAI name in educational materials.

## Uses That Require Explicit Permission

You must ask for permission before doing any of the following:

1. **Naming your product "AUGMANITAI-something"** — For example, "AUGMANITAI Pro", "AUGMANITAI Cloud", "AUGMANITAI for Enterprise". These imply an official relationship.

2. **Using the AUGMANITAI logo or visual marks** (if any) in a promotional or commercial context beyond fair use.

3. **Claiming endorsement** — You cannot imply that AUGMANITAI, Andreas Ehstand, or any project maintainer endorses your product or service without an explicit written agreement.

4. **Registering domain names** that contain AUGMANITAI and that could be confused with the official project (e.g., `augmanitai.com`, `augmanitaipro.io`). If you own such a domain and are not affiliated with the project, please contact the maintainer.

5. **Registering the AUGMANITAI name as a trademark** in any jurisdiction. Do not do this without written permission.

## Prohibited Uses

The following uses are **never** permitted, with or without permission:

1. Using the AUGMANITAI name in a way that is false, misleading, or creates confusion about the source, sponsorship, or approval of your goods or services.

2. Using the AUGMANITAI name in connection with hate speech, harassment, or other conduct that violates the project's Code of Conduct.

3. Using the AUGMANITAI name in a way that disparages the project, its contributors, or its users.

4. Using the AUGMANITAI name in a way that suggests a modified version of AUGMANITAI is the original (this would also violate the CC BY-NC-ND 4.0 NoDerivatives clause applied to the data).

5. Using the AUGMANITAI name to promote activities that contradict the academic mission of the project (pseudoscience, disinformation, commercial scams, etc.).

## Modified Versions

If you fork the AUGMANITAI code under Apache 2.0, your fork must use a different name. You may describe your fork as "a fork of AUGMANITAI" or "based on AUGMANITAI", but you may not:

- Call your fork "AUGMANITAI" or any confusingly similar variant
- Distribute modified terminology data under the AUGMANITAI name (this is additionally prohibited by CC BY-NC-ND 4.0)
- Use the AUGMANITAI logo or visual marks in your fork's branding

When describing your fork, you should make clear that:
1. It is a fork, not the original
2. The original is maintained by Andreas Ehstand
3. Your fork's modifications are your own responsibility

## Fair Use Examples

### Acceptable
- A research paper titled "Extending AUGMANITAI with Memory Terms" that cites the original and proposes additions as a separate contribution
- A blog post titled "Why I Love AUGMANITAI for AI Auditing"
- A GitHub project named "awesome-augmanitai" that curates tools and resources related to AUGMANITAI
- A product description that says "Our tool integrates with AUGMANITAI to provide terminology lookup"
- A conference talk titled "AUGMANITAI: Ontologies for Human-AI Interaction"

### NOT Acceptable
- A commercial product called "AUGMANITAI Enterprise" sold without permission
- A website at `augmanitai.ai` that appears to be the official site but is not
- A GitHub repository named `augmanitai` that hosts a modified version of the terminology
- A social media account named @AUGMANITAI operated by someone not affiliated with the project
- A conference paper that claims to be an "official update" to AUGMANITAI without maintainer involvement

## Reporting Trademark Misuse

If you believe someone is misusing the AUGMANITAI name in violation of this policy, please report it through the private contact channel in `../governance/SECURITY.md`. Include:

1. The URL or other location where the misuse is occurring
2. A description of the misuse
3. Your assessment of whether it is accidental or intentional
4. Any context about the party involved

The maintainer will review the report and respond with the appropriate next steps, which may include a friendly note, a cease-and-desist letter, or in severe cases legal action.
